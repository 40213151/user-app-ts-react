# 職務経歴書用サマリー

## 【履歴書用】簡潔版（1段落）

AIを活用したチャット統合型ヘルプデスク・チケット管理システム「freee aiヘルプデスク」の開発に従事。SharePoint、Google Drive、LINE WORKS、Garoonなど10種類以上の外部サービス連携機能の設計・実装を担当し、OAuth2.0認証フローやAPI統合を含む複雑な連携処理を実装。また、LLMを活用したチケットアシスタント機能のプロンプトエンジニアリングによる回答精度向上、ライセンス管理・アクセス制御（LBAC）の実装、CloudFront経由の大容量ファイルアップロード機能など、バックエンドを中心にフルスタックで開発を推進。技術スタックはRuby on Rails、React/TypeScript、GraphQL、PostgreSQL、OpenSearch、AWSを使用。100件以上のPR実装を通じて、セキュリティ、パフォーマンス、UXの向上に貢献。

---

## プロジェクト概要

**freee aiヘルプデスク** - AIを活用したチャット統合型ヘルプデスク・チケット管理システムの開発に従事。チャットツールから直接問い合わせを受け付け、AIが過去の対応履歴やナレッジベースから最適な回答を自動提案することで、カスタマーサポート業務の効率化と品質向上を実現するSaaS型プロダクト。

**技術スタック**: Ruby on Rails / React / TypeScript / GraphQL / Relay / PostgreSQL / OpenSearch / AWS

**期間**: 2025年10月〜現在

**役割**: フルスタックエンジニア（バックエンド中心）

---

## 主な開発実績

### 1. 外部サービス連携機能の設計・実装（コネクタ開発）

多様なナレッジ管理ツールやチャットツールとの連携機能を、OAuth2.0認証フローやAPI統合を含めて設計・実装。

- **SharePoint連携**: ドキュメントライブラリ・リスト・添付ファイルの同期機能を実装。文字エンコーディング問題やURL正規化の課題を解決
- **Google Drive連携**: ファイル同期機能とOAuth同意フロー実装
- **Google Sites連携**: サイトコンテンツの取得・同期機能実装
- **Google Chat連携**: チャットボット統合とイベント処理実装
- **LINE WORKS連携**: Bot APIを活用したメッセージング機能の実装
- **Garoon（サイボウズ）連携**: セッション管理と同期再開機能の実装
- **Notion連携**: APIエラーハンドリングとデバッグ機能の強化

**技術的ポイント**: 各サービスの認証方式（OAuth2.0、APIキー）の実装、トークンリフレッシュ処理、レート制限対応、エラーリトライ戦略の設計

### 2. AIプロンプトエンジニアリングと回答品質向上

LLM（大規模言語モデル）を活用したチケットアシスタント機能の精度向上に貢献。プロンプト設計の改善により、回答品質を継続的に向上。

- 固有名詞の表記揺れ防止のためのfew-shot example追加
- 回答不可能な質問の判定精度向上
- AIが実行不可能なアクションを約束しない制約の追加
- 質問の言い回しの違いに柔軟に対応するプロンプト改善
- JSON形式パース処理の実装とMarkdownコードブロック削除機能追加
- ハイブリッド回答（中途半端な回答）の防止

**技術的ポイント**: プロンプトエンジニアリング、LLMの挙動制御、構造化出力の実装

### 3. ライセンス管理とアクセス制御（LBAC）の実装

セキュアなマルチテナントSaaSの基盤機能を実装。

- Ebis（ライセンス管理システム）との連携実装
- LBAC（License-Based Access Control）による事業所単位のアクセス制御
- アクティベーション時のライセンス事前検証機能
- Sekisyo（認可システム）との統合

**技術的ポイント**: マルチテナントアーキテクチャ、認可制御、外部システム連携

### 4. パフォーマンス最適化とスケーラビリティ向上

大容量データ処理とファイルアップロードの最適化を実施。

- CloudFront経由のダイレクトアップロード機能実装（大容量ファイル対応）
- GraphQL Mutationのチャンク分割によるタイムアウト対策
- ナレッジドキュメントのファイル数制限撤廃
- OpenSearchインデックス処理のリトライ戦略最適化

**技術的ポイント**: CDN活用、署名付きURL生成、チャンク分割アップロード、非同期処理

### 5. UI/UX改善とオンボーディング体験の向上

ユーザーの初期セットアップ体験を向上させる機能を実装。

- セットアップToDoプログレスバー機能実装
- 各種コネクタの設定ヘルプページ実装（Slack、Google Drive、SharePoint、LINE WORKS、Garoon、Notionなど）
- アクティベーション画面のUI改善とライセンス事前検証機能
- フィードバック詳細画面のUX改善

**技術的ポイント**: React/TypeScript、Relay、UIコンポーネント設計

### 6. セキュリティとエラーハンドリングの強化

本番環境の安定稼働のための品質向上施策を実施。

- DirectUploadのファイル検証強化（セキュリティ対策）
- Bugsnagへのエラーコンテキスト送信機能追加
- リトライ処理の最適化（resque-scheduler導入、リトライ間隔・回数調整）
- 各種エッジケースのエラーハンドリング実装

**技術的ポイント**: セキュリティベストプラクティス、監視・ロギング、冪等性担保

### 7. 運用改善とモニタリング

本番運用を見据えた監視・分析基盤の整備。

- Datadog日次メトリクス送信機能実装
- 教師データ同期のスケジュール最適化
- 各種エラーのBugsnag通知除外設定
